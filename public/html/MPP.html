Постановка задачи

Задание выполняется в группе.

Используя лекционные материалы по данной теме, реализовать доработку презентации по терминам методологии SCRUM. Требуется использовать стратегию «Agile + promotion», то есть задействуя при разработке инструменты управления проектами (в нашем случае — это Trello и инструменты Github), для усовершенствования/доработки презентации по терминам методологии Scrum.

Содержание работы

  1. 1. Создание нового слайда

    Каждый участник группового проекта (внутри своей группы) должен создать новую ветвь (branch) и в ней с помощью кода создать новый слайд (или усовершенствовать, если слайд уже есть в презентации). После этого, участник должен зафиксировать свой вклад в главной ветке (master), создав соответствующее «предложение о слиянии кода» (pull-request). Как только это будет сделано, в интерфейсе GitHub будет создана соответствующая сущность, которую должен подтвердить другой участник этого группового репозитория или сам автор.

    После этого код будет соединен в главной ветке master и переключившись на эту ветку необходимо проверить результаты работы, убедиться, что презентация продолжает работать и отображается корректно.

  2. 2. Внесение доработок и предложений по улучшению

    Во второй части задания требуется посмотреть всю презентацию и по результатам создать одну заявку (issue) с предложением об улучшении какого-либо её аспекта, дополнении слайда, оптимизации структуры или чего-то другого. Максимально подробно и четко описать что необходимо сделать (вашу issue могут выбрать для реализации другие участники проекта).

    После этого, требуется выбрать из всего списка предложений (issues) одну и реализовать предложение, которое в ней содержится. Для этого требуется также как и в пункте 1 задания создать новую ветку (указав в её названии номер выбранной для доработки issue) и в этой ветке реализовать требуемое. После этого повторить процедуру слияния кода с основной веткой (master).

  • • выбор дня и потока выступления;
  • • загрузка презентации;
